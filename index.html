关于nacos集群的相关操作

发表于 2021-01-08

怎么配置nacos

1.预备环境准备

Nacos 依赖 Java 环境来运行。如果您是从代码开始构建并运行Nacos，还需要为此配置 Maven环境，请确保是在以下版本环境中安装使用:
1.64 bit OS，支持 Linux/Unix/Mac/Windows，推荐选用 Linux/Unix/Mac。

2.64 bit JDK 1.8+；

3.Maven 3.2.x+；

2.standalone模式启动

1.打开我们安装的nacos-server文件夹

2.进入bin目录

3.在地址栏中输入cmd

4.进入cmd命令窗口

5.输入  startup.cmd -m standalone

6.在浏览器中输入 http:127.0.0.1:8848

3.配置我们的nacos中的mysql数据库服务

1.打开我们的数据库图形化软件新建一个数据库为nacos(可以随意创建,博主只是懒得改nacos的配置文件)

2.进入我们nacos-server文件夹的conf目录,打开里面的nacos-mysql.sql文件,打开复制粘贴到数据库图形化软件中运行

3.打开conf目录中的application.properties文件,找到关于数据库配置的地方,将其注释打开并修改你的数据库username和password(记得保存)

4.开始搭建nacos集群

1.将我们的nacos-server文件复制为三份以上(因为搭建集群最少为三个服务器)

2.修改application.properties文件里面的nacos服务启动的端口(复制的其他两份也需要我们手动改变nacos服务启动的端口)
(个人建议:将几个端口设置为相邻的,如:8848,8849,8850)

3.将我们conf里面的cluster.conf.example文件删除.example后缀
打开修改里面的ip地址为你自己要访问的ip
(如:127.0.0.1:8848)
(注意:ip和端口之间只有一个:没有空格)

4.(参考standalone模式启动)
命令行输入: startup.cmd -m cluster
(修改了几个nacos服务就可以启动几个)

5.进入我们nacos服务页面,点击左下方的集群管理,查看里面的集群节点(你开启了几个里面就可以有几个集群节点)

注:修改端口的时候一定要查看一下自己配制了哪些服务端口,避免端口重复

5.关于博主遇到的一些坑

启动报错—–一个无法获取数据源的异常

原因:是我们在配置application.properties的时候没有将关于mysql的配置没有打开注释
可以参考博主写的第三点

在查看nacos服务中查看集群的时候只有他自己本身的节点,查看不到其他的节点

原因:是我们再配置cluster.conf的时候ip和端口之间有空格存在
